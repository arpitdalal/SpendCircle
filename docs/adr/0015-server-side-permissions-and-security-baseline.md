# Server-side permissions and security baseline

Spend Circle enforces authentication, Circle membership, ownership, and action permissions inside Convex functions rather than trusting client UI state. Sensitive actions such as Invitations, Feedback, and email sends should be rate-limited where practical: unique Invitation emails are not artificially capped per Circle, resending the same Invitation is capped at 3 per day per Circle/email, total invitation emails are capped at 100 per User per day, Feedback is capped at 20 submissions per User per day, Welcome email sends once per User, and failed Invitation Link attempts are throttled without revealing whether an email belongs to a User. Histories are written server-side only, and secrets live only in platform environment variables.
