# Owner-only invitee email in Circle History

Circle History keeps Invitation actions visible to current Members for access-governance transparency, but invitee email is Owner-only data. Invitation history events (`member invited`, `invitation resent`, `invitation revoked`, and archive/delete-driven revokes surfaced the same way) may store the email in the immutable audit for the current Owner; `listCircleHistory` returns that `email` change only when `resolveCircleAccess` reports `isOwner` at read time. Non-Owners still see the Invitation action row, but the `email` change is omitted entirely — no masked email, placeholder, or "hidden" copy. Accepted Invitations do not change the rule: non-Owners never see the original invite email in older `member invited` events; the accepted person is represented as a Member by Display Name. Ownership transfer follows current authority: the new Owner gains Invitation email details; the former Owner loses them.

Redaction is server-side in `listCircleHistory` only. Writers keep recording `{ field: "email", from | to: <email> }` unchanged; `toHistoryEventView` is shared with Transaction and Category History and is not modified. The web `HistoryList` label map stays as-is — when the API omits the change, the Email row simply does not render.
